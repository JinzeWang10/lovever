# 爱了吗 (Lovever) — 商业计划书

---

## 1. 执行摘要

**爱了吗 (Lovever)** 是一款恋爱承诺登记与历史透明化查询平台，定位为恋爱世界的「天眼查」。

**解决的问题**：恋爱关系中的信息不对称——人们在开始一段感情时，无法可靠地了解对方的感情历史，导致大量本可避免的情感伤害。

**产品机制**：情侣双方在平台双向确认关系承诺，系统永久记录关系历史。任何人在开始新恋情前，可查询对方的定情记录（次数、时间线、状态），实现恋爱信息的透明化。

**市场定位**：全新品类。现有恋爱类App（恋爱记录、情侣签、Lovewick等）均为"当前情侣关系管理工具"，无任何产品触及"恋爱历史透明化查询"领域。爱了吗是市场上第一个恋爱信用基础设施产品。

**商业模式**：前期免费积累用户和数据，后期通过增值服务（详细报告、认证标识、精美证书）和B端合作（婚恋平台数据对接）实现盈利。

**核心壁垒**：数据网络效应——用户越多、记录越多、查询越有价值，后来者无法追赶先行者的数据积累。

## 2. 市场分析

### 2.1 目标市场规模

| 层级 | 定义 | 规模估算 |
|------|------|---------|
| **TAM（总可及市场）** | 中国18-35岁有恋爱需求的人群 | 约3亿人 |
| **SAM（可服务市场）** | 活跃使用社交/恋爱类App的人群 | 约1亿人 |
| **SOM（可获得市场）** | 前3年可触达的早期采纳用户 | 500万-1000万人 |

**市场规模依据**：
- 中国18-35岁人口约3.2亿（国家统计局数据）
- 恋爱类App月活用户超过5000万
- 婚恋社交市场规模超过80亿元/年，且持续增长

### 2.2 用户画像

#### 核心用户群：大学生与初职场人群（18-25岁）

- 恋爱频率高，平均每2-3年一段关系
- 社交媒体重度用户，乐于分享恋爱状态
- 对仪式感需求强烈（官宣、纪念日等）
- 对新鲜事物接受度高，传播意愿强

#### 次核心用户群：职场人士（25-35岁）

- 对恋爱关系更加慎重，有"查一查"的需求
- 更注重信息透明度和安全感
- 付费意愿和能力较高
- 可能面临相亲场景，背景信息需求更强

### 2.3 市场痛点验证

| 痛点 | 验证信号 |
|------|---------|
| 恋爱对象的历史信息不透明 | 社交媒体上大量"如何了解对方过去"的讨论 |
| 隐瞒感情状态/同时交往多人 | "渣男/渣女"话题长期热搜，社会关注度极高 |
| 缺乏承诺仪式感 | 朋友圈"官宣"文化盛行，定情仪式需求旺盛 |
| 恋爱关系缺乏可信凭证 | 婚恋市场背调服务兴起，年增长率超过30% |

## 3. 竞争格局分析

### 3.1 现有恋爱类App定位

| 产品 | 定位 | 与爱了吗的区别 |
|------|------|---------------|
| 恋爱记录 | 当前情侣的纪念日/聊天管理 | 仅服务当前关系，无历史记录功能 |
| 情侣签 | 情侣互动小游戏 | 娱乐导向，无关系登记功能 |
| Lovewick | 情侣关系增进工具 | 关系改善建议，无历史查询 |
| 小恩爱 | 情侣社交空间 | 私密空间定位，分手后数据消失 |

**关键发现**：所有现有恋爱类App都是"当前关系管理工具"，仅服务于正在恋爱中的情侣，且关系结束后产品价值归零。**没有任何产品触及恋爱历史的记录和查询领域。**

### 3.2 间接替代方案

| 替代方式 | 局限性 | 爱了吗的优势 |
|---------|--------|-------------|
| 朋友圈官宣 | 可删除、可设置不可见、无法验证 | 永久记录、不可篡改 |
| 口头承诺 | 无第三方见证、无法追溯 | 系统化记录、双向确认 |
| 朋友打听 | 信息碎片化、覆盖面窄 | 结构化数据、一键查询 |
| 婚恋网站背调 | 成本高（数百-数千元）、仅婚恋阶段 | 免费/低成本、覆盖所有恋爱阶段 |

### 3.3 爱了吗的差异化壁垒

1. **品类定义者**：首创"恋爱信用查询"品类，先入者优势巨大
2. **数据网络效应**：每新增一个用户和记录，都增加平台对所有用户的查询价值
3. **双边锁定**：关系记录涉及双方，数据无法迁移到竞品
4. **社会心智占领**：率先建立"恋爱前查一查"的用户习惯

### 4.2 版本迭代路线图

#### V1.0 — 核心验证（0-6个月）

- 完成MVP核心功能开发
- 小范围内测（1000-5000用户）
- 验证核心假设：用户是否愿意登记关系并查询他人
- 收集用户反馈，优化核心流程

#### V2.0 — 体验升级（6-12个月）

- 定情证书美化（多种模板、自定义设计）
- 恋爱时间线可视化
- 关系纪念日提醒
- 社交分享优化（分享卡片、小程序链接）
- 增值服务上线（详细报告、认证标识）

#### V3.0 — 生态拓展（12-24个月）

- 恋爱信用评分体系
- B端API开放（婚恋平台接入）
- 关系咨询/调解服务入口
- 跨平台数据打通

## 5. 商业模式

### 阶段一：完全免费，积累用户和数据（0-12个月）

**策略**：所有核心功能完全免费，最大化用户增长和数据积累。

**目标**：
- 注册用户达到100万+
- 累计定情记录50万+
- 建立足够密度的数据网络

**收入**：无（或少量广告测试）

### 阶段二：增值服务变现（12-24个月）

| 增值服务 | 定价参考 | 目标转化率 |
|---------|---------|----------|
| 详细历史报告（包含更多维度信息） | 9.9-29.9元/次 | 5-10% |
| "已认证"标识（身份信息真实性认证） | 49.9元/年 | 3-5% |
| 精美定情证书（设计师模板、实体邮寄） | 19.9-99.9元 | 10-15% |
| 恋爱时间线精美页面 | 9.9元/年 | 5-8% |
| VIP会员（查询次数不限 + 所有增值功能） | 98元/年 | 2-5% |

**预估收入**：假设100万活跃用户，综合ARPU 5-10元/年，年收入500-1000万元。

### 阶段三：B端合作与生态变现（24个月+）

| 合作方向 | 模式 | 潜在收入 |
|---------|------|---------|
| 婚恋平台数据对接 | API调用计费 | 按查询量收费 |
| 恋爱信用评分授权 | 授权使用费 | 年度授权费 |
| 婚庆/珠宝品牌合作 | 联合营销/导流 | CPS/广告费 |
| 情感咨询服务导流 | 佣金分成 | 交易分成 |

## 6. 增长策略

### 6.1 社交裂变机制

定情行为天然具备社交传播属性：

1. **定情链接传播**：一方发起定情，对方必须下载App确认 → 自然拉新
2. **证书分享裂变**：定情证书设计精美，用户主动分享到朋友圈/社交媒体
3. **查询驱动增长**：A想查B的记录，需要注册才能查 → 查询需求拉新
4. **话题传播**：每一次定情、每一次分手都是社交话题的素材

**核心裂变公式**：每对情侣定情 = 至少2个注册用户 + N个朋友圈曝光 + M个好奇查询

### 6.2 内容营销

| 渠道 | 策略 | 内容方向 |
|------|------|---------|
| 小红书 | KOL合作 + UGC运营 | "定情证书晒照"、"恋爱前查一查" |
| 抖音 | 话题挑战 + 情感类短视频 | #爱了吗挑战#、定情仪式感vlog |
| 微博 | 话题运营 + 热搜借势 | 情感热点关联讨论 |
| B站 | 情感类UP主合作 | 深度内容："恋爱该不该透明？" |

### 6.3 冷启动策略

**第一阶段：种子用户（0-1万用户）**

- 面向大学校园推广，从1-2所大学的情侣群体开始
- 与校园情侣相关社团/活动合作
- 邀请制注册，制造稀缺感

**第二阶段：校园扩张（1万-10万用户）**

- 覆盖更多高校，建立校园大使体系
- 举办校园"定情日"活动
- 联合校园表白墙/匿名社区推广

**第三阶段：破圈增长（10万-100万用户）**

- 社交媒体内容营销全面铺开
- KOL/明星合作扩大知名度
- 产品口碑传播 + 自然增长

## 7. 风险与应对

### 7.1 隐私合规风险

| 风险点 | 应对策略 |
|-------|---------|
| 个人恋爱信息属于敏感数据 | 严格遵守《个人信息保护法》，仅在用户授权下采集和展示 |
| 历史记录被恶意利用 | 信息分级展示：公开信息仅限概况（次数/时间），详细信息需授权 |
| 用户要求删除个人信息 | 设计合规的数据处理机制，平衡记录完整性与个人权利 |
| 未成年人保护 | 注册实名认证 + 年龄门槛（18岁以上） |

**法律合规路径**：
- 双方定情属于双方共同产生的关系数据，非单方面隐私
- 公开查询仅展示脱敏概况，不泄露具体身份信息
- 咨询法律顾问，确保产品设计符合现行法规

### 7.2 用户增长瓶颈

| 瓶颈 | 应对策略 |
|------|---------|
| 早期数据稀疏，查询无结果 | 前期强化"定情仪式感"价值，降低对查询功能的依赖 |
| 鸡生蛋问题（没人查→没人登记） | 从校园熟人社交圈切入，保证局部数据密度 |
| 单身用户没有使用动机 | 开放查询功能给所有用户，单身者可查询潜在恋爱对象 |

### 7.3 道德争议应对

可能面临的争议及品牌立场：

| 争议 | 品牌立场 |
|------|---------|
| "这是不是在监控恋爱？" | 我们记录的是双方自愿做出的承诺，不是监控 |
| "过去的恋爱凭什么被查？" | 信息对称是健康关系的基础，我们只展示概况而非细节 |
| "这会不会导致恋爱歧视？" | 我们提供信息，不做判断；恋爱历史本身不是缺点 |
| "分手后记录无法删除不公平" | 关系记录是双方共同产生的客观事实，类似企业经营记录 |

**核心叙事**：爱了吗不做道德判断，只提供信息透明。我们相信知情权是健康关系的基础。
