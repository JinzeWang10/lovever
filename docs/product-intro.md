# 爱了吗 (Lovever) — 产品介绍文档

> **一句话定位**：恋爱世界的「天眼查」——让每一段感情都有据可查

---

## 1. 产品概述

**爱了吗 (Lovever)** 是一款恋爱承诺登记与历史查询平台。情侣在平台上双向确认关系承诺，系统永久记录关系历史。任何人在开始新恋情前，可以查询对方的历史定情记录，了解对方的感情经历概况。

品牌名「Lovever」取自 Love + Forever，寓意每一段真心的承诺都值得被记录和尊重。

**产品性质**：全新品类——恋爱信用基础设施
**核心类比**：天眼查记录企业信用，爱了吗记录恋爱信用

---

## 2. 问题与机会

### 恋爱中的信息不对称

在恋爱关系中，"对方的感情历史"是一个极其重要却几乎无法验证的信息维度：

- **信息获取靠自述**：对方谈过几段恋爱、每段持续多久、为何结束，完全依赖对方的口头陈述，无法验证真伪
- **隐瞒成本极低**：隐藏一段过去的关系，或者同时维持多段关系，在技术上毫无门槛
- **信息不对称造成伤害**：不知道对方是否已有伴侣、是否频繁更换关系、是否有过不忠记录，导致许多人在感情中受到本可避免的伤害

### 现有方案的不足

| 现有方式 | 局限性 |
|---------|--------|
| 口头询问 | 完全靠自觉，无法验证 |
| 朋友圈/社交媒体 | 可删除、可隐藏，不完整 |
| 朋友打听 | 信息碎片化，覆盖面窄 |
| 婚恋网站背调 | 只在婚恋阶段介入，且成本高 |

### 市场机会

- 中国18-35岁适龄恋爱人群超过3亿
- 恋爱类App市场规模持续增长，但无任何产品触及"关系历史透明化"这一方向
- 社会对感情诚信的需求日益增长，但缺乏系统化的解决方案

---

## 3. 解决方案

爱了吗通过三个核心机制解决恋爱信息不对称问题：

### 机制一：双向确认的关系登记

- 一方发起定情邀请，对方确认后关系正式记录
- 双方必须使用手机号实名验证，确保身份唯一性
- 系统记录关系的建立时间、双方信息和当前状态

### 机制二：不可篡改的历史记录

- 所有定情记录永久保存，不可单方面删除
- 关系状态变化（在一起 → 已分手）全程留痕
- 形成每个人完整的恋爱历史时间线

### 机制三：新关系建立前的历史披露

- 发起新的定情邀请时，系统自动向双方展示彼此的历史记录
- 在充分知情的前提下决定是否确认新关系
- 实现"定情前尽职调查"

## 4. 核心功能

### 4.1 定情承诺

情侣双方在平台进行关系登记，形成一份双向确认的"定情记录"。

- **发起定情**：一方输入对方手机号，发起定情邀请
- **对方确认**：对方收到通知，确认同意后关系正式建立
- **电子凭证**：生成定情证书，记录双方姓名、日期和承诺
- **状态管理**：关系进行中 / 已分手，状态变更需当事人操作

### 4.2 历史查询

任何注册用户可以通过手机号查询一个人的恋爱历史概况。

- **查询入口**：输入对方手机号
- **返回信息**：历史定情次数、每段关系的时间范围和状态
- **隐私保护**：仅展示关系概况，不展示对方伴侣的具体身份信息（除非双方已建立定情关系）
- **信息分级**：基础信息免费可查，详细报告作为增值服务

### 4.3 定情前披露

在建立新关系前，系统自动展示双方的历史记录，实现信息对称。

- **自动触发**：发起定情邀请时自动生成双方历史报告
- **双向展示**：双方都能看到对方的完整历史
- **知情决策**：在充分了解的前提下决定是否建立新关系
- **降低风险**：有效减少因信息不对称造成的感情伤害

## 5. 用户故事

### 故事一：小美的新恋情

> 小美在社交场合认识了一个男生，双方互有好感，开始约会。男生提议在爱了吗上"定情"，小美搜索了男生的手机号，发现他有过2段恋爱记录，最近一段3个月前结束，持续了1年半。小美觉得这是正常的感情经历，放心地确认了定情。

**价值体现**：在开始正式关系前获取了可靠的感情背景信息，做出了知情决策。

### 故事二：阿强避开了一个"渣男"

> 阿强的朋友小林开始和一个女生交往，好友们在爱了吗上查询了对方，发现她在过去2年内有5段定情记录，平均每段不到5个月。朋友们把这个信息告诉了小林，小林更谨慎地对待了这段关系。

**价值体现**：恋爱历史的透明化帮助用户识别潜在风险信号。

### 故事三：情侣间的信任仪式

> 大学情侣阿杰和小雨恋爱半年后，决定在爱了吗上"定情"。他们把定情证书截图发到朋友圈，收到了一百多个赞。对他们而言，爱了吗的定情不仅是信息登记，更是一种浪漫的承诺仪式。

**价值体现**：定情行为本身具有仪式感和社交传播价值。

## 7. 差异化优势

### 为什么市场上没有替代品？

| 维度 | 现有恋爱App | 爱了吗 (Lovever) |
|------|-----------|-----------------|
| **定位** | 当前关系管理工具 | 恋爱信用基础设施 |
| **时间维度** | 只关注"现在" | 记录过去，服务未来 |
| **用户范围** | 仅服务当前情侣 | 服务所有恋爱中/即将恋爱的人 |
| **核心价值** | 增进感情互动 | 恋爱信息透明化 |
| **数据价值** | 关系结束后无价值 | 历史数据持续产生价值 |
| **网络效应** | 弱（仅双人使用） | 强（用户越多，查询越有价值） |

### 核心壁垒

1. **品类先发优势**：全新品类，先入者定义规则
2. **数据网络效应**：用户和数据越多，查询价值越高，后来者难以追赶
3. **社会共识建设**：率先建立"恋爱前查一查"的用户心智
4. **双边锁定效应**：关系记录涉及双方，迁移成本极高
